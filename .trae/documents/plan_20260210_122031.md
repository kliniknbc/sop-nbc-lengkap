Pesan error `Command "npm run build" exited with 126` di Vercel berarti **Permission Denied** (Izin Ditolak).

Penyebab utamanya adalah **folder `node_modules` ikut ter-upload ke GitHub**.
Karena Anda meng-upload dari Windows, file eksekusi di dalamnya (seperti `vite`) formatnya berbeda dengan yang dibutuhkan server Vercel (Linux). Akibatnya, Vercel gagal menjalankannya.

### Solusi
Saya akan membuat file `.gitignore` yang terlewat, agar folder sampah/sistem tidak ikut ter-upload.

Setelah itu, Anda perlu menjalankan beberapa perintah Git untuk membersihkan repository Anda.

### Langkah Perbaikan:
1.  **Saya akan membuat file `.gitignore`** yang berisi daftar folder yang harus diabaikan (`node_modules`, `dist`, dll).
2.  **Anda perlu menjalankan perintah ini** di terminal Anda (setelah saya selesai) untuk menghapus `node_modules` dari GitHub tanpa menghapusnya dari komputer Anda:

```bash
git rm -r --cached node_modules
git rm -r --cached dist
git add .gitignore
git commit -m "fix: remove node_modules and add gitignore"
git push
```

Setelah Anda push, Vercel akan otomatis mendeteksi perubahan, melakukan instalasi ulang yang benar, dan error 126 akan hilang.
