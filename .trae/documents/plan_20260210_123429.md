User ingin menambahkan penanganan **Error State** pada aplikasi React, kemungkinan karena saat ini jika API gagal, aplikasi hanya diam atau menampilkan data kosong tanpa pesan error.

Saya akan melakukan refactoring pada `src/services/api.js` dan komponen-komponen UI untuk menangani error dengan benar.

### Rencana Implementasi

1. **Update** **`src/services/api.js`**:

   * Mengubah fungsi API agar **melempar (throw) error** saat fetch gagal, bukannya mengembalikan array kosong/objek default secara diam-diam.

   * Ini memungkinkan komponen UI mengetahui kapan terjadi error.

2. **Update Komponen UI (`LoginModal`,** **`Operasional`,** **`Keuangan`)**:

   * Menambahkan state baru: `const [error, setError] = useState(null)`.

   * Menggunakan blok `try-catch` saat memanggil API.

   * Menampilkan pesan error di UI (misalnya teks merah atau alert) jika `error` state terisi.
